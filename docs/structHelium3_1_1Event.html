Detailed Description

Encapsulates a machine’s state change in the simulation.

Each Event represents a transition of a machine (e.g., a truck or station) into a new state—such as mining, moving, arriving, or unloading—starting at a specific time and lasting for a given duration. Events are processed in the simulation loop and recorded by the ILog implementation for activity logging.

In the simulation’s priority queue, Events are ordered by their end time (start + duration), with shorter durations winning ties.
